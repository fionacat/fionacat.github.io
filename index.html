leetcode-647：Palindromic Substrings

发表于 2019-04-20 | 分类于 leetcode

题目

给定一个字符串，您的任务是计算该字符串中有多少回文子字符串。
具有不同起始索引或结束索引的子字符串被计为不同的子字符串，即使它们包含相同的字符。
例1：

输入：“ABC”
产量：3
说明：三个回文串：“A”、“B”、“C”。

例2：

输入：“AAA”
产量：6
说明：六个回文串：“A”、“A”、“A”、“AA”、“AA”、“AAA”。

思路（Manacher算法）

  1. 在每两个字符（包括头尾）之间加没出现的字符（如%），这样所有字符串长度就都是奇数
  2. 记录p[i]表示i能向两边推（包括i）的最大距离
  3. 假设p[1~i-1]已经求好了，现在要求p[i]：
  4. 假设当前能达到的最右边为R，对应的中点为pos，j是i的对称点。

    当i＜R时

  5. 情况1

    由于L~R是回文，所以p[i]=p[j]（i的最长回文和j的最长回文相同）。
  6. 情况2

    j的最长回文跳出L。那么i的最长回文就不一定是j的最长回文了，但蓝色的肯定还是回文部分，所以p[i]>R-i,剩下部分暴力破解
    总结
    p[i]=min(p[2*pos-i],R-i)

    2.当i>=R时

    由于后面是未知的，于是只能暴力处理了。

思路（暴力优化）

对于一个长度为n的字符串，其回文子串的中心位置可能出现的2 * n - 1个位置（可能出现在字符上也可能出现在两个字符的中间）
对于每个中心位置，如果[a,b]是回文，则[a+1,b-1]也是回文

算法

对于每个中心位置，向两边扩展字符串，判断新生成的字符串其是否为回文
扩展条件为left >= 0 && right < s.length() && s[left] == s[right]

leetcode-338：Counting Bits

发表于 2019-04-20 | 分类于 leetcode

题目

给定一个非负整数num。对于0≤i≤num范围内的每个数字i，计算其二进制表示形式中的1个数，并将其作为数组返回。
例1：

输入：2
输出：[01,1,1]

例2：

输入：5
输出：[0,1,1,2,1,2]

要求： 使用运行时o（n*sizeof（integer））很容易找到解决方案。但你能在线性时间o（n）/可能在一次通过中做到这一点吗？ 空间复杂性应该是O（N）。 你能像个老板那样做吗？不使用任何内置函数，如C++中的Bug BuffTynPopCube或任何其他语言。

思路1

对于任意一个数n，bit位上1的个数，取决于n/2上1的个数和n的末位是否为1，则有公式ret[n] = ret[n/2] + (n & 1 == 1 ? 1 : 0)

代码

思路2

首先是一个数减1，对应二进制的变化就是最右的一个1变为0，而这个1右边的所有0变为1，即相当于包括最后一个1在内的右边所有位取反，例如12（1100）减1，得到11（1011），然后再与变化前的数12（1100）进行与&运算，得到8（1000），可以看出经过这样一个运算之后这个数的1的个数减少了一个，所以可以利用这个原理，得到res[i] = res[i & (i -1 )] + 1

leetcode-198：House Robber

发表于 2019-04-19 | 分类于 leetcode

题目

你是一个计划沿着街道抢劫房屋的专业抢劫犯。每栋房子都有一定数量的钱被藏起来，唯一阻止你抢劫它们的限制是相邻的房子都有安全系统连接，如果两个相邻的房子在同一个晚上被闯入，它会自动联系警察。
给出一个非负整数的列表，代表每个房子的钱的数量，确定你今晚可以抢劫的最大金额，而不需要报警。
例1：

输入：[1,2,3,1]
产量：4
说明：Rob House 1（钱=1），然后Rob House 3（钱=3）。
您可以抢劫的总金额=1+3=4。

例2：

输入：[2,7,9,3,1]
产量：12
说明：Rob House 1（金钱=2）、Rob House 3（金钱=9）和Rob House 5（金钱=1）。
您可以抢劫的总金额=2+9+1=12。

思路1

对于当前房子 i , 有两种选择：

  • 偷，累加i-2之前所偷的赃物
  • 不偷，累加i-1之前所偷的赃物

根据这两种情况，可以判断哪种选择所偷的赃物多，进行选择，则有dp[i] = max(dp[i-2]+nums[i],dp[i-1)

  • 累加i-2之前所偷的赃物
  • 累加i-3之前所偷的赃物
    根据这两种情况，可以判断哪种选择所偷的赃物多，进行选择，则有dp[i] = max(dp[i-3]+nums[i],dp[i-2]+nums[i]）

leetcode-438：Find All Anagrams in a String

发表于 2019-04-19 | 分类于 leetcode

题目

给定一个字符串s和一个非空字符串p，在s中查找p的变位词的所有起始索引。
字符串仅由小写英文字母组成，字符串s和p的长度将不大于20100。
输出顺序无关紧要。
例1：

输入：
S：“cbaebabacd”P：“ABC”
输出：
〔0, 6〕
说明：
起始索引为0的子字符串是“cba”，它是“abc”的变位词。
起始索引为6的子字符串是“bac”，它是“abc”的变位词。

例2：

输入：
S：“ABAB”P：“AB”
输出：
〔0, 1, 2〕
说明：
起始索引为0的子字符串是“ab”，它是“ab”的变位词。
start index=1的子字符串是“ba”，它是“ab”的变位词。
起始索引为2的子字符串是“ab”，它是“ab”的变位词。

思路

首先，先构造一个map，对于p中的每个字符char，都有map[char]++。
然后，初始化一个长度为0的窗口，left = 0，right = 0。第一步先扩展窗口，也就是在right的右边界上做文章。每次right读到一个字符char，都有map[char]–。当map[char]的值大于等于1时，很明显就是窗口中进入了一个p中含有的字符。我们可以取一个变量count，值为p中所有字符的总数。每次有一个p中字符进入窗口，则count–。这样，当count == 0的时候，表明我们的窗口中包含了p中的全部字符，得到一个结果。
当窗口包含一个结果以后，为了进一步遍历，我们需要缩小窗口使窗口不再包含全部的p，同样，如果map[char]>=0，表明一个在p中的字符就要移除窗口，那么count ++，以此类推。
